StatEval: A Comprehensive Benchmark for Large Language Models in Statistics

Paper GitHub

StatEval, developed by the team of Professor Fan Zhou at Shanghai University of Finance and Economics, is the first benchmark systematically organized along both difficulty and disciplinary axes to evaluate large language models’ statistical reasoning.

It includes a Foundational Knowledge Dataset comprising exactly 22,262 problems (9,382 undergraduate and 12,880 graduate instances) curated from 76 classical textbooks and extensive exam collections.

Furthermore, it features a Statistical Research Dataset consisting of 84,179 proof-based tasks derived from 6,953 high-impact research articles (published between 2000 and 2025). These tasks are categorized by derivation difficulty into 40,366 Easy, 22,013 Medium, and 21,800 Hard problems.

Overview

A comprehensive breakdown of the first dedicated benchmark for statistical reasoning in LLMs

1. Data Processing Pipeline

To overcome the brittleness of pure rule-based methods and the hallucination risks of pure LLM approaches, the StatEval pipeline adopts a synergistic hybrid modular architecture. By strategically combining deterministic regular expressions for structural extraction with the powerful semantic orchestration of GPT-5.4 and GPT-5.4-mini, the end-to-end workflow comprises six sequential modules:

1 File-Conversion

Homogenizes PDFs, scanned manuscripts, and LaTeX sources into structured Markdown. Employs advanced MLLMs (e.g., MinerU) for high-fidelity mathematical OCR and layout reconstruction.

2 Context-Segmentation

Isolates theorems and their surrounding contexts using a hybrid Regex+LLM engine. It expertly untangles nested "umbrella" proofs, extracts block equations, and compiles a Global Notation Reference Table.

3 Dependency Parsing

Establishes the logical topology between theoretical elements. It precisely maps internal cross-theorem links, non-theorem structural premises (e.g., Assumptions, Definitions), and external literature citations.

4 Pair Synthesis

Generates self-contained theorem-proof setups. Employs a strict Context-Aware Patching mechanism and a Zero-Deletion Policy to expand derivation steps safely without losing original textual integrity.

5 QA Generation

Scales problems into Easy, Medium, and Hard variants based on the dependency graph. Features a Non-Refine Mode for absolute text fidelity and a Refine Mode for LLM-smoothed, standardized logical flow.

6 Quality Validation

Embeds secondary LLM audits to rigorously check proof completeness, logical sufficiency, and difficulty consistency, complemented by a Human-in-the-Loop feedback system to optimize upstream prompts.

2. Evaluation Protocol

To ensure rigorous and interpretable evaluation, StatEval implements distinct scoring mechanisms tailored to problem structures, powered by GPT-5.4-mini as the judging model. While Multiple-Choice Questions are graded via exact matching (0 or 1), Open-Ended questions utilize an Adaptive Process-Based Scoring Pipeline.

Phase 1: Logic Alignment & Routing

Mathematical proofs often allow for multiple valid solution paths. To avoid unfairly penalizing correct but non-standard derivations, a Selector LLM compares the student's problem-solving logic with the reference solution. The response is categorized as either Aligned (following the reference strategy) or Divergent (employing an alternative method). This ensures the response is routed to the most appropriate evaluation mechanism.

Phase 2A: Granular Step-Based Verification (For Aligned Paths)

For strategically aligned responses, the system emulates human expert grading by decoupling the assessment into Decomposition and Evaluation. The reference is parsed into a 10-point Grading Rubric of atomic logical steps. The rubric adapts based on problem difficulty:

  • Easy: Focuses directly on the terminal proof process and logical transitions.
  • Medium: Allocates 50% weight to the derivation of prerequisite lemmas, and 50% to the primary research result.
  • Hard: Evaluates the entire independent reasoning chain without pre-defined weight splits, emphasizing global coherence.

Phase 2B: Independent Path Verification (For Divergent Paths)

For valid alternative methodologies, the reference solution is bypassed. The system strictly verifies if preconditions for invoked theorems are met (penalizing forced conclusions) and evaluates the custom trajectory across two core dimensions:

  • Logical Coherence (Slogic): Validity of the overarching proof strategy and step transitions.
  • Technical Precision (Stech): Algebraic and analytical accuracy of intermediate derivations.
